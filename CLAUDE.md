# CLAUDE.md

## 铁律：对齐颗粒度

这个项目的每一页都是一个**AI驱动的生成工具**，不是静态参考文档。

- `/topics` = 输入赛道 → AI生成选题
- `/script` = 输入选题 → AI路由判定 → 管线执行 → 输出脚本
- `/positioning` = 输入信息 → AI生成定位

**每一页的交互模式**：用户输入 → 调用AI → 流式输出结果。

当我被要求"把XX系统加入网页分身"时，我要做的是：
1. 先读现有页面的代码模式（`useGeneration` hook、API路由、输入输出格式）
2. 按同样的模式构建新页面——输入表单 + AI调用 + 结果展示
3. **不是建一个静态参考文档页面**

## 剧情种草系统的正确落地方式

`/seeding` 页面应该像 `/script` 一样：
- 输入：产品、脸谱、发愿、人群
- 调用：`buildSeedingPrompt()` → AI生成
- 输出：完整的剧情种草脚本（按时间轴：画面/台词/时长）

不是展示框架和技法的参考页。那是md文件的事。网页是用来跑的。
